StringTable Sample
     General Information

Syntax: dataID, string
Where dataID must be an integer and the comma (,) is required.


The precompile resource file (filename.RC) must include the text of the string and it must fit on a single line of 255 characters maximum.  Therefore, there cannot be any LineFeeds, CarriageReturns or Quotation Marks.  Use inserts and replace them.

Some possible inserts when saving strings:
\n = Quote
\c = CarriageReturn
\l = LineFeed
%s = variable string to be replaced at runtime

Format for the precompile resource file (filename.RC) entry:

STRINGTABLE
BEGIN
  101, "Resource Editor"
END
Then to Call from program...
Me.Caption = LoadResString(101)


As you can see from the above, using strings for resources in vb isn't very practical - except for multi-language support in applications.  Take particular notice of the sequential nature of the dataID.
// ENGLISH
STRINGTABLE
BEGIN
  101,  "Yes"
  102,  "No"
  103,  "Cancel"
END

// SPANISH
STRINGTABLE
BEGIN
  201,  "Si"
  202,  "No"
  203,  "Cancele"
END


In the program...
lngEnglish = 100
lngSpanish = 200

varLanguage = lngEnglish

cmdYes.Caption = LoadResString(1 + varLanguage)
cmdNo.Caption = LoadResString(2 + varLanguage)
cmdCancel.Caption = LoadResString(3 + varLanguage)
